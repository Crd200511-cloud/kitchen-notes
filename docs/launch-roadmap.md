# 给别人使用的上线路线图

## 阶段 1：可内测

目标：

- 让少量真实用户开始使用

范围：

- 网站上线
- 用户登录
- AI 整理
- 食谱保存 / 编辑 / 删除
- 基础额度限制

你要完成：

1. 部署前端
2. 部署后端
3. 接入数据库
4. 配置环境变量
5. 绑定正式域名

## 阶段 2：可邀请

目标：

- 可控地开放给更多用户

范围：

- 搜索
- 导入记录
- AI 调用日志
- 错误告警
- 限流

你要完成：

1. 增加日志表
2. 增加用量统计页
3. 增加错误监控
4. 增加管理员查看能力

## 阶段 3：可平台化

目标：

- 不只给用户直接使用，也给别人接入

范围：

- 对外 API
- SDK
- 接入文档
- 合作方密钥管理

你要完成：

1. 抽出公共 AI 服务
2. 设计合作方 API Key
3. 提供调用文档
4. 增加合作方配额控制

## AI 插件方案的正确顺序

不要一开始就做插件。

正确顺序：

1. 先把网站做成正式产品
2. 再把 AI 服务抽成统一接口
3. 再做 SDK
4. 最后再做插件

## 你当前最值得优先完成的任务

1. 数据库表和接口落地
2. 登录和用户数据隔离
3. AI 次数限制
4. 正式部署

## 建议的下一步实现顺序

1. 先做数据库
2. 再做登录
3. 再把本地存储改成云端
4. 再补完整的列表和详情接口
5. 最后加合作方接入能力
